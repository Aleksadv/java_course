# Практика 1 - Спецификаторы доступа внутренних классов

**Public внутренний класс:**
- Доступен отовсюду
- Можно создавать из любого кода: `outer.new PublicInner()`

**Protected внутренний класс:**
- Доступен в том же пакете и наследникам
- Из другого пакета доступен только наследникам внешнего класса

**Package-private внутренний класс (без спецификатора):**
- Доступен только в том же пакете
- Не доступен из других пакетов, даже наследникам

**Private внутренний класс:**
- Доступен только во внешнем классе
- Нельзя создать извне внешнего класса

# Практика 2 - Доступ внутреннего класса к внешнему

**Внутренний класс имеет доступ ко ВСЕМ полям и методам внешнего класса**, включая private. Не зависит от спецификаторов доступа.

Пример:
```java
class Outer {
    private String privateField = "private";
    
    class Inner {
        void access() {
            System.out.println(privateField); // Доступно!
        }
    }
}
```

# Практика 3 - Доступ внешнего класса к внутреннему
**Внешний класс имеет доступ только к public элементам внутреннего класса.** Для доступа к private/protected полям нужны геттеры.

Пример:
```java
class Outer {
    class Inner {
        private String secret = "secret";
        public String info = "info";
    }
    
    void test() {
        Inner inner = new Inner();
        System.out.println(inner.info);    // Доступно
        // System.out.println(inner.secret); // Ошибка!
    }
}
```
