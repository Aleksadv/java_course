# final в блоке catch

## Что означает:
```java
catch (final Exception e) {
    ...
}
```

## Зачем нужно:
- Запрещает переназначение переменной исключения
- Предотвращает случайные изменения ссылки на исключение
- Улучшает читаемость - показывает, что переменная не меняется

**Пример ошибки без final:**
```java
catch (Exception e) {
    e = new Exception("новое сообщение"); // Изменение ссылки
    // Оригинальное исключение потеряно!
}
```
**С final:**
```java
catch (final Exception e) {
    // e = new Exception(); // ОШИБКА КОМПИЛЯЦИИ!
    // Можно использовать e, но нельзя переназначать
    System.out.println(e.getMessage());
}
```
**Особенности:**
- Не влияет на само исключение (оно immutable)
- Только запрещает изменение ссылки на исключение
- Рекомендуется для лучшего кода
